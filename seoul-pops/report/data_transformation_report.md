- **자료형:** `float64`(29개), `int64`(3개)
- **컬럼 수:** 32개
- **행(Row) 수:** 305,280 개
- **메모리 사용량:** **약 74.5 MB**

### 📌 변환 후 (`.parquet` 로드 직후)
- **자료형:** `float64`(2개), `int32`(2개), `int8`(1개), `category`(2개)
- **컬럼 수:** 7개 (`기준일ID`, `시간대구분`, `행정동코드`, `총생활인구수`, `생활인구수`, `성별`, `연령대`)

- **메모리 사용량:** **약 220.1 MB**

> [!TIP]
> **데이터 구조 및 용량 변화 분석**
> 1. **Tidy-data 변환:** 32개의 넓은 컬럼을 녹여내면서(Melt) 행(Row)의 수가 약 **28배(854만 줄)** 증가했습니다. 이는 Tidy 형태의 자연스러운 결과입니다.

>   - 문자열(성별, 연령대)을 `category` 타입으로 변환
>   - 정수형 데이터를 `int64`에서 `int32` 및 `int8` 등으로 다운캐스팅

## 3. Parquet 메타 정보(Metadata) 분석

Parquet 형식은 빅데이터 처리에 최적화된 컬럼 기반(Columnar) 저장 포맷입니다. 저장된 최종 파케이 파일의 내부 메타데이터를 확인한 결과는 다음과 같습니다.

### 📌 메타 정보 요약

```text
  created_by: parquet-cpp-arrow version 24.0.0
  num_columns: 6
  num_rows: 8547840
  num_row_groups: 9
  format_version: 2.6
  serialized_size: 9734
```

### 📌 Row Group 정보 (예시)
```text
  num_columns: 6
  num_rows: 1048576
  total_byte_size: 5969217
```

> [!TIP]
> **메타 정보의 의미**
> - **num_row_groups (9개):** Parquet는 거대한 데이터를 파일 하나에 통째로 저장하는 대신, 여러 개의 행 그룹(Row Group) 단위로 분할하여 저장합니다. 이를 통해 데이터를 필터링해서 읽을 때, 전체 파일을 읽지 않고 조건에 맞는 그룹만 선택적으로 스캔(Predicate Pushdown)할 수 있어 속도가 매우 빠릅니다.
> - **num_rows (8,547,840개):** 전체 저장된 데이터 행의 개수입니다.
> - **total_byte_size (약 6MB):** 1개의 Row Group에는 약 104만 개의 행이 저장되어 있으며, 각각이 고도로 압축되어 6MB라는 아주 작은 크기를 차지하고 있음을 알 수 있습니다.
> - **serialized_size (9734 바이트):** 이 메타데이터 자체를 저장하는 용량입니다. 메타데이터에는 각 컬럼의 데이터별 최소값/최대값, Null 개수 등이 포함되어 데이터 분석 도구가 데이터를 스마트하게 쿼리할 수 있도록 돕습니다.
