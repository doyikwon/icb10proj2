# EDA 및 ML 마크다운 리포트 생성 가이드

이 스킬은 주어진 데이터셋에 대해 탐색적 데이터 분석(EDA)과 머신러닝(ML) 모델링을 수행하고, 그 결과를 종합적인 마크다운 아티팩트로 출력하는 방법을 정의합니다.

## 핵심 요구사항 (필수 준수)

### 1. 분석 파이프라인 시각화 (Mermaid)
- 리포트 서두에 전체 데이터 분석 및 모델링 과정(데이터 전처리 -> EDA -> 피처 엔지니어링 -> 모델 학습 -> 평가)을 설명하는 상세한 `Mermaid` 워크플로우 시각화를 포함해야 합니다.
- 오류 없이 렌더링되도록 괄호 등의 특수문자 사용에 주의하여 작성하세요.

### 2. 시각화 및 통계표 구성
- **시각화 도구**: 반드시 `matplotlib` 및 `seaborn`을 사용하세요.
- **한글 폰트**: `koreanize-matplotlib` 모듈을 import하여 한글 폰트가 깨지지 않도록 설정하세요.
- **표 필수 포함**: 모든 그래프 아래에는 해당 그래프를 설명하는 수치형 표(막대그래프는 피봇테이블/교차표, 히스토그램이나 박스플롯은 기술통계표 등)를 마크다운 표 형식으로 함께 제공해야 합니다.
- **시각화 저장**: 에이전트는 파이썬 스크립트를 작성하여 로컬(아티팩트 폴더 등)에 이미지를 `.png`로 저장하고, 마크다운 리포트에서 `![caption](/absolute/path/to/image.png)` 형식으로 불러와야 합니다.

### 3. 머신러닝 평가 (3개 모델, 5개 지표)
분류(Classification) 문제를 기본으로 하며, 다음을 반드시 포함하세요:
- **사용 모델 (3개 이상)**: Random Forest, Gradient Boosting, XGBoost (필요 시 LightGBM 등 추가 가능).
- **평가 지표 (5개 이상)**: Accuracy, Precision, Recall, F1-Score, ROC-AUC.
- 각 모델의 평가 지표를 비교하는 통합 표와, Feature Importance 또는 Confusion Matrix 시각화 및 관련 통계표를 포함해야 합니다.

### 4. 심층 인사이트 (섹션별 500자 이상)
가장 중요한 부분입니다. 분석 결과(수치형 변수 탐색, 범주형 변수 탐색, 퍼널 분석 등)와 모델링 결과에 대해 단순한 현상 나열이 아닌, **해석과 실무적 적용 방안을 포함한 500자 이상의 상세한 인사이트**를 각 단계별로 작성해야 합니다.
(예: 특정 피처가 타겟 변수에 미치는 영향을 분석하고, 이를 활용한 마케팅 또는 비즈니스 의사결정 전략 제안)

## 동작 방식 (Execution Workflow)

1. **파이썬 스크립트 작성**: 에이전트는 주어진 데이터셋을 불러오고, 위 요구사항에 맞게 EDA와 모델링을 수행하며, 시각화 이미지를 저장하고 마크다운 리포트를 자동 생성하는 파이썬 스크립트를 작성합니다.
2. **스크립트 실행**: 작성한 파이썬 스크립트를 백그라운드 태스크나 `run_command`로 실행합니다.
3. **리포트 통합 및 작성**: 파이썬 스크립트가 실행을 마치면, 추출된 통계표와 저장된 이미지 경로, 에이전트가 직접 분석한 '심도 있는 인사이트(500자 이상)'를 융합하여 단일 마크다운 아티팩트 파일(예: `report.md`)로 출력합니다.
4. (선택사항) 만약 파이썬 스크립트가 직접 마크다운 리포트 텍스트까지 모두 파일로 생성하도록 작성했다면, 해당 파일을 아티팩트로 복사하여 사용자에게 제공합니다. 단, 인사이트가 매우 상세해야 하므로 에이전트가 LLM의 추론 능력을 살려 직접 인사이트를 작성해 넣는 방식을 권장합니다.

## 팁
- 스크립트 실행 중 에러가 발생하면, 로그를 확인하고 스크립트를 수정한 뒤 재실행하세요.
- 데이터의 타겟(Target) 변수를 명확히 파악하고 전처리(결측치, 스케일링, 인코딩)를 정확히 수행하세요.
